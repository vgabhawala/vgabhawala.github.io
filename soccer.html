Desktop Soccer Game

Project Timeline: March 2024 – May 2024

This project aimed to create a desktop-scale soccer game that combines mechatronics, mechanical design and user interactivity. The game simulates a mini soccer field where a user-controlled kicker competes against a moving goalkeeper, mounted on a linear actuator. It provided a hands-on opportunity to integrate design, fabrication, and programming skills in a playful, engaging format.

Components

  • 1× Arduino Uno
  • 1× HiLetgo L298N Motor Driver
  • 1× NEMA11 stepper motor Linear Rail Actuator
  • 1× Gikfun 2" 4Ω 3W Speaker
  • 1× 3D Printed Flexible Kicker Mechanism
  • 1× Laser-Cut Game Base and Structure
  • 1× LED Display
  • 1× Goal Frame with printed goalkeeper
  • 1× Breadboard
  • Jumper wires, resistors, capacitors
  • Power supply (5V for logic, 12V for actuator)

Ideation

The original goal for the desktop soccer game was to create a fully portable and compact system that could be easily set up and played anywhere. To test this, I initially constructed a standalone base using cardboard. By using the desk itself as the foundation for the game, I eliminated the need for a bulky base and reduced the total number of components, resulting in a cleaner, more minimal footprint.

Design Process

The final design featured a 3D printed kicking mechanism and a ball holder base to keep the ball in place before each kick. After testing various materials, TPU (Thermoplastic Polyurethane) was chosen for its flexibility and durability. The kicker’s infill density was reduced to enhance its springiness and improve shot power. The goal was also 3D printed and designed to align with the actuator-driven goalkeeper. Field boundaries were laser cut from acrylic, allowing for quick assembly and a clean, portable design.

Electronics Added

Video 1. Prototype Tested

To support two-player gameplay, the system included an electronic scoreboard powered by an Arduino Uno and displayed on an LCD screen. A custom 3D printed enclosure housed a set of mechanical buttons that allowed players to manually increase their score or reset it at any time. The buttons were wired to the Arduino, which updated the screen in real time. The modular casing made it easy to service or modify the electronics, and the Arduino-based system allowed for future expansion—such as adding automatic scoring or sound feedback.

To add a dynamic challenge to the game, a linear actuator was installed behind the goalpost, controlled by a motor driver and programmed to move the goalkeeper back and forth at random intervals. This movement made scoring more difficult and added an unpredictable, interactive element to each play. The actuator was synchronized with the rest of the system using the Arduino, ensuring reliable and responsive motion.
